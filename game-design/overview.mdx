# Game Overview

Temporal Rift is a competitive–cooperative strategy game for **3–5 players**, each acting as a time-traveller aligned with a hidden faction. Players observe future events before they occur and take actions in earlier timelines to influence or prevent those outcomes. Conflicting actions create paradoxes that must be resolved before the timeline can advance.

## Design intent

Every core mechanic is intentionally modelled after **event-driven microservices architecture**.

| Game concept | Architecture analogue |
|---|---|
| Future events | Event logs / projections in an event-sourced system |
| Past actions | Commands that mutate state and may trigger new events |
| Paradoxes | Race conditions or conflicting writes |
| Paradox resolution | Conflict resolution strategies in distributed systems |
| Timeline advancement | Advancing to the next event window, updating shared state |

## Aggregates

| Aggregate | Owner | Key invariants |
|---|---|---|
| `Lobby` | game-service / session | 3–5 players, all factions assigned, no duplicates |
| `Game` | game-service / session | Cascade counter triggers collapse at 3; era counter triggers stabilization at 5 |
| `ActionRound` | game-service / action | One action per player; round closes on all-submitted or timer |
| `FutureEvent` | timeline-service | Probabilities sum to 100%; ceiling 90%, floor 0%; annihilated outcome cannot win |
| `PlayerState` | game-service (split) | Hand ≤ 5 cards; score ≥ 0; faction immutable once assigned |
| `WeaverChain` | timeline-service | Links reference real past outcomes; one active chain per Weaver |
